# 코딩 컨벤션

> 프로덕션 코드를 작성·수정하는 모든 에이전트(programmer, refactorer)가 따르는 공통 규약.

## 네이밍
- 클래스: PascalCase
- 함수/변수: camelCase
- 상수: UPPER_SNAKE_CASE
- 파일명은 주요 export 이름과 일치시킨다.

## 주석
- "무엇을" 이 아니라 "왜" 를 적는다.
- 자명한 코드에는 주석을 달지 않는다.
- TODO 주석에는 담당 이슈/근거를 함께 남긴다.

## 에러 처리
- 시스템 경계(외부 API, 사용자 입력)에서만 방어적으로 검증한다.
- 내부 함수 호출에는 일어날 수 없는 케이스를 위한 방어 코드를 넣지 않는다.
- 예외를 삼키지 말고, 의미 있는 컨텍스트와 함께 re-throw 한다.

## 함수 / 모듈
- 단일 책임. 함수가 너무 길면 쪼갠다.
- 인자가 4개를 넘으면 객체/구조체로 묶는 것을 고려한다.
- 순수 함수를 우선한다.

## 금지 사항
- 추측성 추상화(미래를 대비한 설정 플래그, 헬퍼) 금지.
- 미사용 코드, 데드 코드 커밋 금지.
