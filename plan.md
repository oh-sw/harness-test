# Plan

## Overview
웹브라우저에서 동작하는 도트(픽셀 아트) 스타일 2P 로컬 횡스크롤 1:1 격투게임을 구현한다. 순수 HTML/CSS/JS(ES modules) 기반으로, 캔버스에 픽셀 아트로 렌더링하고 키보드 입력으로 조작한다. 전체 UI(타이틀, 버튼, 액자, HP 바, 폰트 느낌)는 일관된 픽셀 아트 스타일을 따른다(이미지 스무딩 off, 정수 스케일). 화면은 캐릭터 선택 → 대전 → 결과 순서로 전환되며, 캐릭터 5명(현재/재현/상우/한결/중훈)과 체력 10, 펀치/킥 데미지 1, 킥은 버튼 입력 후 0.5초 준비자세를 거쳐 발동, 펀치는 앞쪽만 막기 가능, 킥은 막기 불가 규칙을 따른다. 테스트는 Vitest(jsdom)로 순수 로직 단위 테스트를 작성한다.

## Commit 1: 프로젝트 셋업 및 정적 엔트리 (픽셀 아트 베이스)
- **목표**: 브라우저에서 열 수 있는 최소 HTML/JS 프로젝트 구조와 테스트 환경 구축, 픽셀 아트 렌더링 베이스(캔버스 `image-rendering: pixelated`, `ctx.imageSmoothingEnabled=false`, 정수 스케일) 설정.
- **변경 대상**:
  - `index.html` (canvas + 화면 컨테이너, 픽셀 폰트 느낌 스타일 적용)
  - `src/main.js` (엔트리, 현재는 픽셀 아트 스타일 "FIGHTING GAME" 타이틀만 표시)
  - `src/styles.css` (픽셀 아트 톤: 검정 배경, `image-rendering: pixelated`, monospace/픽셀 폰트 계열)
  - `package.json` (vitest, jsdom 개발 의존성, `test` 스크립트)
  - `vitest.config.js`
  - `tests/smoke.test.js`
- **구현 메모**: 빌드 도구 없이 native ES modules 사용. 캔버스 논리 해상도는 낮게(예: 320x180) 잡고 CSS 로 정수배 확대. 테스트는 Node 환경 Vitest.
- **테스트**: `tests/smoke.test.js` 에서 `src/main.js` 의 상수(`GAME_TITLE`) import 확인.
- **완료 조건**: `npm test` 통과, `index.html` 을 브라우저로 열면 픽셀 스타일 타이틀이 또렷한 도트로 보임(안티 에일리어싱 없음).
- **Use cases (QA용)**:
  1. `index.html` 을 브라우저로 열면 픽셀 아트 스타일의 "FIGHTING GAME" 타이틀이 화면에 표시된다.
  2. 확대해서 봐도 글자/그래픽이 흐릿하지 않고 또렷한 도트 형태로 보인다.
  3. 콘솔에 에러가 없다.

## Commit 2: 캐릭터 데이터 및 선택 화면 (픽셀 얼굴 + 액자) + 해상도 업스케일
- **목표**: 5명 캐릭터(현재/재현/상우/한결/중훈) 정의와 캐릭터 선택 화면(1P/2P 선택 → START 버튼) 구현. 각 캐릭터는 **랜덤하게 생성된 남성 얼굴 픽셀 아트**를 가지며, 선택 화면에서는 픽셀 아트 **액자(frame)** 안에 해당 남성 얼굴 초상화가 그림으로 표시된다. 또한 전체 캔버스 논리 해상도를 Commit 1 의 320x180 에서 **1280x720** 으로 상향하고, CSS 스케일/표시 크기도 이에 맞게 재조정한다. 1P 와 2P 가 **같은 캐릭터를 선택할 수 없으며**, 시도 시 3초 지속 경고 모달을 표시한다.
- **변경 대상**:
  - `src/main.js` (캔버스 논리 해상도 320x180 → 1280x720 변경, 관련 상수 수정, 스크린 라우팅 init)
  - `src/styles.css` (CSS 스케일/크기 조정: 논리 1280x720 에 맞춰 표시 배율 재계산)
  - `index.html` (canvas width/height 속성 1280x720 으로 변경, 선택 화면 컨테이너)
  - `src/characters.js` (id, name, 그리고 결정적 랜덤(시드 기반) 남성 얼굴 생성용 파라미터: 피부톤/헤어스타일/헤어컬러/눈/수염 등. 5명: 현재/재현/상우/한결/중훈)
  - `src/art/face.js` (시드 → 남성 얼굴 픽셀 아트를 캔버스에 그리는 함수. 픽셀 단위 그리기, 남성 스타일 프리셋만 사용)
  - `src/art/frame.js` (픽셀 아트 액자 렌더링: 테두리/장식 도트)
  - `src/screens/select.js` (선택 UI: 각 캐릭터를 액자+얼굴+이름 카드로 표시, 1P/2P 선택 하이라이트, 중복 선택 방지 로직 및 경고 모달)
  - `tests/characters.test.js`
- **구현 메모**:
  - **해상도 변경**: Commit 1 에서 설정한 캔버스 논리 해상도 320x180 과 CSS 스케일을 이 커밋에서 1280x720 으로 변경한다. `main.js` 의 `CANVAS_WIDTH`/`CANVAS_HEIGHT` 상수, `index.html` 의 canvas 태그 width/height 속성, `styles.css` 의 캔버스 표시 크기(CSS width/height 또는 scale factor)를 모두 수정해야 한다. 기존 Commit 1 코드에서 320/180 을 참조하는 모든 곳을 1280/720 으로 갱신. 논리 해상도 1280x720 이므로 CSS 표시 크기는 1280x720 그대로 1배로 표시하거나, 뷰포트에 맞춰 적절한 배율을 적용한다.
  - 각 캐릭터는 고정 시드를 가져 매 로드 시 동일 얼굴이 재현되지만(결정적), 캐릭터 간에는 서로 다른 랜덤 남성 얼굴이 나오도록 한다.
  - 얼굴은 작은 해상도(예: 32x32) 도트로 그려 액자 안 캔버스(또는 서브 캔버스)에 렌더.
  - 액자는 픽셀 테두리(2~3px 도트)로 구성, 선택 상태에 따라 색상이 달라짐(1P=파랑, 2P=빨강 테두리).
  - 선택 상태 `{p1: id, p2: id}`, START 시 `game:start` 커스텀 이벤트 발행.
  - **중복 선택 방지**: 1P 가 이미 선택한 캐릭터를 2P 가 선택하려 하거나 그 반대인 경우, 선택을 무시하고 **3초 지속 경고 모달**을 화면 중앙에 표시한다. 모달은 픽셀 아트 스타일(픽셀 테두리, 도트 폰트)이며 "같은 캐릭터를 선택할 수 없습니다" 메시지를 담는다. 3초 후 자동으로 사라진다. 모달이 떠 있는 동안에도 다른 캐릭터로의 선택 변경은 가능하다.
- **테스트**:
  - 캐릭터 목록이 5명이고 이름이 "현재/재현/상우/한결/중훈" 과 정확히 일치.
  - 각 캐릭터가 얼굴 생성용 시드/파라미터를 가지고 있으며, 같은 시드로 호출 시 동일한 픽셀 데이터가 생성됨(결정적).
  - 선택 상태 reducer 함수의 동작 검증(1P/2P 선택 변경).
  - 중복 선택 시 reducer 가 선택을 거부하고 경고 플래그를 반환하는지 검증.
- **완료 조건**: 테스트 통과 + 캔버스 논리 해상도가 1280x720 으로 동작하며 브라우저에서 5개의 액자 카드가 픽셀 아트로 표시되고 각 액자 안에 서로 다른 남성 얼굴 도트 그림이 보이며, 1P/2P 선택에 따라 액자 테두리 색이 바뀌고, 같은 캐릭터 중복 선택 시 경고 모달이 3초간 표시됨.
- **Use cases (QA용)**:
  1. 페이지 로드 시 픽셀 아트 액자 5개가 나란히 표시되고, 각 액자 안에 서로 다른 남성 얼굴 도트 그림과 "현재/재현/상우/한결/중훈" 이름표가 보인다.
  2. 캔버스 논리 해상도가 1280x720 으로 렌더링되어, Commit 1 대비 더 세밀한 픽셀 표현이 가능하며 여전히 안티 에일리어싱 없이 또렷하다.
  3. 같은 캐릭터는 페이지를 새로고침해도 동일한 얼굴로 보인다(결정적 랜덤).
  4. 1P 칸에 "현재"를, 2P 칸에 "재현"을 선택하면 해당 액자 테두리가 1P/2P 색으로 강조되고, START 를 누르면 대전 화면으로 전환된다.
  5. 1P 가 "현재"를 선택한 상태에서 2P 도 "현재"를 선택하면, 선택이 반영되지 않고 화면 중앙에 픽셀 아트 스타일 경고 모달("같은 캐릭터를 선택할 수 없습니다")이 나타난다.
  6. 경고 모달은 3초 후 자동으로 사라지며, 모달이 표시되는 동안에도 다른 캐릭터로 선택을 변경할 수 있다.

## Commit 3: 대전 화면 기본 - 픽셀 스프라이트 렌더링과 좌우 이동/점프
- **목표**: 캔버스에 두 파이터를 픽셀 아트 스프라이트로 렌더링(선택 화면의 얼굴을 몸체 위에 얹음), 횡스크롤 격투 스테이지 배경(픽셀 아트), 좌우 이동 및 점프(중력) 구현.
- **변경 대상**:
  - `src/engine/fighter.js` (상태: x, y, vy, hp 등)
  - `src/engine/physics.js` (중력, 지면 충돌, 경계 제한)
  - `src/engine/input.js` (keydown/keyup, 키맵 1P/2P)
  - `src/art/body.js` (픽셀 아트 몸체 스프라이트 + 얼굴 합성)
  - `src/art/stage.js` (픽셀 아트 배경: 바닥/하늘/장식)
  - `src/screens/battle.js` (게임 루프, 렌더)
  - `tests/physics.test.js`, `tests/fighter.test.js`
- **구현 메모**:
  - 키맵 1P: f(left) t(jump) h(right) g(block) z(punch) x(kick)
  - 키맵 2P: ArrowLeft/Up/Right/Down, `,` punch, `.` kick
  - 렌더링은 전부 픽셀 단위(정수 좌표), 스무딩 off.
  - **동시입력 지원**: 1P와 2P가 동시에 키를 눌러도 각각 독립적으로 동작해야 한다. 예를 들어 1P가 h를 누른 상태에서 2P가 ArrowUp을 누르면 1P는 오른쪽으로 이동하면서 2P는 점프한다. 각 키의 눌림 상태를 독립적으로 추적하여 프레임마다 모든 눌린 키에 대한 동작을 동시에 처리해야 한다.
  - **한글/영어 키입력 둘 다 지원**: 한글 입력 모드(IME)에서도 키 입력이 정상 동작해야 한다. 예를 들어 1P의 'h' 키는 한글 모드에서 'ㅗ'로 입력되더라도 오른쪽 이동이 동일하게 동작해야 한다. `event.key` 대신 `event.code` 기반으로 키를 판별하여 IME 상태와 무관하게 물리 키 위치로 동작하도록 한다.
- **테스트**: 점프 시 vy 감소→지면복귀, 좌/우 입력에 x 변화, 경계 clamp, 동시입력 시 두 플레이어 상태가 독립적으로 갱신됨, 한글 키코드(event.code 기반) 입력 시에도 동일하게 동작함.
- **완료 조건**: 테스트 통과 + 브라우저에서 픽셀 아트 배경과 두 픽셀 파이터가 이동/점프하며, 각 파이터의 얼굴이 선택 화면에서 본 그 남성 얼굴과 동일. 1P와 2P가 동시에 움직일 수 있으며, 한글 입력 모드에서도 키 입력이 정상 동작.
- **Use cases (QA용)**:
  1. 대전 화면 배경/캐릭터 모두 픽셀 아트 스타일로 또렷하게 보인다.
  2. 각 파이터의 얼굴은 선택 화면 액자에서 본 해당 캐릭터의 얼굴과 동일하다.
  3. 1P 가 `h` 를 누르면 오른쪽으로 이동한다.
  4. 2P 가 `↑` 를 누르면 점프했다가 착지한다.
  5. 1P가 `h`를 누른 상태에서 2P가 `↑`를 누르면 1P는 오른쪽으로 이동하면서 동시에 2P는 점프한다(두 입력이 서로 간섭하지 않는다).
  6. 한글 입력 모드에서 1P가 'ㅗ'(h 키 위치)를 누르면 영문 모드와 동일하게 오른쪽으로 이동한다.
  7. 한글 입력 모드에서 2P의 방향키(ArrowUp 등)는 영문 모드와 동일하게 동작한다.


## Commit 4: 공격 - 펀치, 킥, 막기, 데미지 로직 (픽셀 HP 바)
- **목표**: 펀치/킥/막기 및 데미지 적용, 킥 0.5초 준비자세 메커니즘, 앞쪽 막기 규칙, 공격 모션 0.5초 유지 및 모션 중 입력 잠금, 막기 홀드 메커니즘, 픽셀 아트 스타일 HP 바 표시.
- **스코프 경계**:
  - 포함: 펀치/킥/막기 판정, 데미지 적용, 킥 준비자세 타이머(0.5초), 공격 모션 유지(0.5초) 및 모션 중 입력 잠금, 막기 홀드, HP 바 HUD
  - 미포함: 승패 판정 및 결과 화면 전환(Commit 6), 액션별 스프라이트 모션(Commit 5)
- **변경 대상**:
  - `src/engine/combat.js` (hitbox, resolveAttack, 모션 타이머, 입력 잠금 판정)
  - `src/engine/fighter.js` (hp, blocking, kickCharging, kickChargeStartTime, attackState, motionTimer, inputLocked)
  - `src/art/hud.js` (픽셀 아트 HP 바, 이름표)
  - `src/screens/battle.js` (HUD 연결)
  - `tests/combat.test.js`
- **테스트**:
  1. 앞쪽에서 펀치 + 막기 → hp 변화 없음
  2. 뒤쪽에서 펀치 + 막기 → hp -1
  3. 킥 + 막기 → hp -1
  4. 킥 버튼 입력 후 0.5초 경과 전에는 킥이 발동되지 않음(준비자세 상태)
  5. 킥 버튼 입력 후 0.5초 경과 시 킥이 자동 발동됨
  6. 펀치/킥 모션(데미지 판정 포함)이 0.5초간 유지됨
  7. 모션 중 이동/점프/막기/추가 공격 입력이 무시됨(입력 잠금)
  8. 모션 0.5초 종료 후 입력 잠금이 해제되어 다시 이동/공격 가능
  9. 막기 버튼을 누르고 있는 동안 막기 상태가 계속 유지됨
  10. 막기 버튼을 떼면 막기 상태가 즉시 해제됨
- **완료 조건**: 모든 combat 테스트 통과 + 브라우저에서 픽셀 HP 바가 감소하는 것을 관찰 가능.
- **Use cases (QA용)**:
  1. 캔버스 상단에 픽셀 아트 스타일의 HP 바(블록형) 두 개가 각 플레이어 이름과 함께 표시된다.
  2. 1P 가 막기(`g`)를 누르고 있는 동안 막기 상태가 유지되며, 2P 펀치(`,`)를 앞에서 맞아도 1P HP 바가 줄지 않는다. 막기 버튼을 떼면 막기가 즉시 해제된다.
  3. 1P 가 막기 중이어도 2P 킥(`.`)을 맞으면 1P HP 바가 한 블록 감소한다.
  4. 2P 가 킥(`.`) 버튼을 누르면 즉시 발차기가 나가지 않고 준비자세를 취하며, 0.5초 후에 발차기가 발동된다.
  5. 킥 준비자세 중에는 아직 킥 판정이 발생하지 않으므로 상대에게 데미지가 들어가지 않는다.
  6. 펀치 또는 킥 발동 후 모션이 0.5초간 유지되며, 이 동안 데미지 판정도 0.5초간 활성 상태이다.
  7. 펀치/킥 모션 0.5초 동안에는 이동, 점프, 막기, 추가 펀치/킥 입력이 모두 무시된다(입력 잠금).
  8. 모션 0.5초가 끝나면 입력 잠금이 풀려 다시 자유롭게 이동/공격할 수 있다.

## Commit 5: 머리 위 점프 넘기 및 공격 방향 제한

- **목표 (What/Why)**: 현재 두 가지 문제가 있다. (1) 두 캐릭터가 서로의 머리 위로 뛰어넘을 수 없다 — `preventOverlap`이 항상 x축 겹침을 밀어내므로 공중에서도 통과가 불가능하다. 격투 게임에서 상대를 뛰어넘어 뒤로 돌아가는 것은 기본 전술이므로 이를 허용해야 한다. (2) 펀치/킥이 방향과 무관하게 사거리 안이면 판정되며, 스프라이트도 항상 한쪽만 바라본다 — 공격은 자기가 보고 있는 방향에 있는 상대에게만 모션과 데미지가 적용되어야 한다.
- **스코프 경계**:
  - 포함: 공중 통과 허용(지상에서만 겹침 방지), 공격 판정에 방향 조건 추가(attacker가 바라보는 방향에 defender가 있을 때만 hit), 스프라이트 좌우 반전(facingRight에 따라 렌더링 flip)
  - 미포함: 자동 페이싱(캐릭터가 자동으로 상대를 바라보는 기능)은 포함하지 않음 — facingRight는 초기값 고정. 새로운 공격 타입이나 데미지 규칙 변경 없음
- **변경 대상**:
  - `src/engine/physics.js` (preventOverlap: 둘 다 지면에 있을 때만 겹침 방지, 한쪽이라도 공중이면 통과 허용)
  - `src/engine/combat.js` (resolveAttack 또는 inRange: attacker의 facingRight 방향에 defender가 있는지 검사하는 조건 추가)
  - `src/art/body.js` (facingRight 파라미터에 따라 스프라이트를 좌우 반전하여 렌더링)
  - `src/screens/battle.js` (drawBody 호출 시 facingRight 전달)
  - `tests/physics.test.js` (점프 넘기 관련 테스트 추가)
  - `tests/combat.test.js` (방향 기반 공격 판정 테스트 추가)
- **테스트**:
  1. 두 파이터가 지면에 있을 때 x축 겹침 시 서로 밀어냄 (기존 동작 유지)
  2. 한쪽 파이터가 공중에 있을 때 x축 겹침이 허용됨 (통과 가능)
  3. 공중 통과 후 착지하면 다시 겹침 방지가 적용됨
  4. attacker가 오른쪽을 보고 있고 defender가 오른쪽에 있을 때 펀치가 적중함
  5. attacker가 오른쪽을 보고 있고 defender가 왼쪽에 있을 때 펀치가 적중하지 않음
  6. attacker가 오른쪽을 보고 있고 defender가 왼쪽에 있을 때 킥이 적중하지 않음
- **완료 조건**: 모든 physics/combat 테스트 통과 + 브라우저에서 점프로 상대 머리 위를 넘을 수 있고, 뒤를 향한 공격이 빗나가는 것을 확인 가능.
- **Use cases (QA용)**:
  1. 1P가 2P 바로 앞에서 점프(`t`)하여 2P의 머리 위를 넘으면, 1P가 2P 뒤쪽에 착지한다 (공중에서 밀려나지 않는다).
  2. 1P가 2P의 왼쪽에서 오른쪽을 바라보며 펀치(`z`)를 누르면 사거리 안의 2P에게 데미지가 적용된다.
  3. 1P가 점프로 2P를 넘어간 뒤(1P가 2P 오른쪽에 위치), 1P가 여전히 오른쪽을 바라보는 상태에서 펀치(`z`)를 누르면 왼쪽에 있는 2P에게 데미지가 적용되지 않는다.
  4. 킥도 동일하게 자기가 바라보는 방향에 있는 상대에게만 데미지가 적용된다.
  5. 스프라이트가 facingRight 값에 따라 좌우 반전되어 렌더링된다 (1P는 오른쪽, 2P는 왼쪽을 바라보는 모습).

## Commit 7: 결과 화면 및 재시작 (픽셀 스타일)
- **목표**: 한쪽 HP 0 도달 시 픽셀 아트 스타일 결과 화면 전환, 승자의 얼굴을 액자에 표시, 재시작.
- **변경 대상**:

  - `tests/result.test.js`
- **구현 메모**: `game:end` 이벤트에 winnerId 전달. 결과 화면은 전체 픽셀 아트 톤 유지, 승자 이름과 액자 속 얼굴 초상화를 크게 표시하며 "RESTART" 버튼 → 선택 화면으로.
